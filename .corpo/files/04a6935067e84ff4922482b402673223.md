---
title: "corpo threads"
description: "List all threads in a corpus file as JSON, each with author, body, replies, and a needsResponse flag. Used by agents to discover which threads require a reply or action."
---

Reads a corpus file and outputs all threads as JSON. Each entry includes the
thread's content and a `needsResponse` flag indicating whether the last message
was authored by someone other than `claude`.

## Invocation

```sh
bun src/index.ts threads <fileId>
```

## Output Shape

```json
[
  {
    "id": "9ac91b73",
    "author": "me",
    "timestamp": "2026-03-09T05:47:16.873Z",
    "body": "...",
    "replies": [
      { "author": "claude", "timestamp": "...", "body": "..." }
    ],
    "needsResponse": false
  }
]
```

`needsResponse: true` when the last author in the thread (root or final reply)
is not `claude`. This is the signal the `resolve-threads` skill uses to identify
actionable threads.
