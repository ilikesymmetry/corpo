## Pending Features

1. **Practice `corpo init` in a new repo (A)** — Validate the init flow end-to-end in a fresh repo. Stress test the UX as a first-time user.

2. **Cross-repo syncing** — After (A) exists, `corpo init` a second repo (B) that depends on (A). Build the sync mechanism so remote files from A are accessible in B.

3. **GUI on GitHub Pages** — Any corpo repo should automatically get a GUI hosted on GitHub Pages, no CLI install required. Enables read-only browsing for anyone.

4. **Dev harness for corpo itself** — Add testing, linting, pre-commit hooks, and docs cleanup tooling to the corpo repo. Makes future iterations safer and faster.

5. **`@claude` comment triggers** — Improve iteration pace by letting authors write a comment with `@claude` to trigger a Claude Code instance that detects it and opens a branch proposal in response.

6. **GUI for PR review** — A GUI view for reviewing the diff between a new branch and main, integrated into the corpo GUI.
