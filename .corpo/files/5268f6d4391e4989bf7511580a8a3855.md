Starts the local GUI server. Binds to `localhost` only. No auth —
access is scoped to the current machine.

## Invocation

```bash
# Foreground (human terminal use — blocks until Ctrl+C)
bun src/index.ts serve [--port 3000]

# Background (agent use — returns immediately)
bun src/index.ts serve --detach [--port 3000]
```

**Detach output:**
```json
{ "url": "http://localhost:3000", "pid": 12345, "detached": true }
```

## Blocking vs. Detached

By default `corpo serve` blocks — it keeps the process alive until Ctrl+C,
which is the expected behavior when run in a terminal. When invoked from an
agent, pass `--detach` to spawn the server as a background process and return
control immediately. The server continues running after the parent exits.

The detached process is re-spawned using `process.execPath` (the bun binary)
and `process.argv[1]` (the entry script), so it works identically in both
dev and installed contexts.

## Root Discovery

Walks up from the current working directory until it finds `.corpo/config.json`
— the same pattern `git` uses to find `.git/`. The discovered root becomes the
base path for all file reads and writes during the session.

## What the Server Does

- **LocalAdapter** reads from `.corpo/files/{id}.md` and writes on commit
- **Hono** handles all routing (REST API + static asset serving)
- **GUI SPA** served from `gui/dist/` for all non-API routes
- In dev, Vite proxies `/api/*` to the Hono server at port 3000

## Implementation

`src/commands/serve.ts` — command definition
`src/server/index.ts` — Hono app factory (`createServer`)
`src/server/adapter.ts` — `LocalAdapter` class
