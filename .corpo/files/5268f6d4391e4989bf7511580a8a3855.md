---
title: "corpo serve"
description: "Start a local Hono HTTP server that serves the Corpo viewer SPA and REST API against the nearest .corpo corpus. Finds corpus root by walking up from CWD. Default port 3000."
---

Starts the local viewer server. Binds to `localhost` only. No auth —
access is scoped to the current machine.

## Invocation

```sh
bun src/index.ts serve [--port 3000]
```

## Corpus Discovery

Walks up from the current working directory until it finds `.corpo/config.json`
— the same pattern `git` uses to find `.git/`. The discovered root becomes the
base path for all file reads and writes during the session.

## What the Server Does

- **LocalAdapter** reads from `.corpo/files/{id}.md` and writes on commit
- **Hono** handles all routing (REST API + static asset serving)
- **Viewer SPA** served from `viewer/dist/` for all non-API routes
- In dev, Vite proxies `/api/*` to the Hono server at port 3000

## Implementation

`src/commands/serve.ts` — command definition
`src/server/index.ts` — Hono app factory (`createServer`)
`src/server/adapter.ts` — `LocalAdapter` class
