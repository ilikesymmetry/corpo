# CLI Stack

The corpo CLI is a TypeScript binary built with incur, compiled with Bun, and
distributed as a single self-contained executable. The GUI SPA is embedded
in the binary at compile time.

See [GUI Stack](b5f3e2d1a4c84f9ab1e7c2d6f3b5a4e8) for the GUI side.

---

## Runtime — Bun

Bun compiles TypeScript to a single binary with no external runtime dependency:

```sh
bun build --compile --minify src/index.ts --outfile corpo
```

Bun is chosen for three reasons specific to corpo:

1. **Asset embedding** — the GUI `dist/` directory is embedded into the
   binary at compile time. Bun handles this natively via `--asset-naming`.
2. **Built-in SQLite** — the cache DB (`~/.corpo/cache.db`) uses
   `bun:sqlite` with no additional dependency.
3. **Built-in HTTP server** — `Bun.serve()` powers `corpo serve` with no
   extra install.

## HTTP Layer — Hono

`corpo serve` exposes the six-endpoint API and serves the embedded GUI.
[Hono](https://hono.dev) handles routing (~14KB, runs on Bun natively):

```ts
import { Hono } from 'hono'
import { serveStatic } from 'hono/bun'

export function createServer(root: string) {
  const app = new Hono()

  app.get('/api/navigation', async (c) => {
    const config = await readConfig(root)
    return c.json(config)
  })

  app.put('/api/navigation', async (c) => {
    const { navigation } = await c.req.json()
    await writeConfig(root, { ...await readConfig(root), navigation })
    return c.body(null, 204)
  })

  app.get('/api/files/:id', async (c) => {
    const raw = await readFile(root, c.req.param('id'))
    if (raw === null) return c.notFound()
    return c.text(raw)
  })

  app.post('/api/files', async (c) => {
    const { raw } = await c.req.json()
    const id = randomUUID().replace(/-/g, '')
    await writeCorpoFile(root, id, raw)
    return c.json({ id }, 201)
  })

  app.put('/api/files/:id/commit', async (c) => {
    const id = c.req.param('id')
    if (!await fileExists(root, id)) return c.notFound()
    const { raw } = await c.req.json()
    await writeCorpoFile(root, id, raw)
    return c.body(null, 204)
  })

  app.delete('/api/files/:id', async (c) => {
    const id = c.req.param('id')
    await deleteFile(root, id)  // handles local + remote + navigation cleanup
    return c.body(null, 204)
  })

  // Serve embedded GUI SPA for all other routes
  app.get('*', serveStatic({ root: './gui/dist' }))

  return app
}
```

## Project Structure

```
packages/cli/
  src/
    index.ts          # CLI entrypoint — Cli.create().command(...).serve()
    commands/
      init.ts         # corpo init
      new.ts          # corpo new [--group]
      serve.ts        # corpo serve [--port] [--detach]
      reply.ts        # corpo reply
      threads.ts      # corpo threads
      lint.ts         # corpo lint
      resolve.ts      # corpo resolve
      mv.ts           # corpo mv
      update.ts       # corpo update
    server/
      index.ts        # createServer(root) — Hono app
      adapter.ts      # LocalAdapter implementation
    lib/
      config.ts       # findRoot, readConfig, writeConfig, nav helpers
      files.ts        # readCorpoFile, writeCorpoFile, createCorpoFile, …
      id.ts           # generateId() → UUID v4, hyphens stripped
  bin/
    corpo.js          # Node shim — calls bun to run src/index.ts
  gui/
    dist/             # Built GUI assets, embedded into binary at compile time
packages/gui/         # GUI SPA (separate Vite project)
```

## Shared Primitives

### Finding the root

Every command walks up from CWD, mirroring how git finds `.git/`:

```ts
import { access } from 'node:fs/promises'
import { dirname, join } from 'node:path'

export async function findRoot(start = process.cwd()): Promise<string> {
  let dir = start
  while (true) {
    try {
      await access(join(dir, '.corpo', 'config.json'))
      return dir
    } catch {
      const parent = dirname(dir)
      if (parent === dir) throw new Error('Not inside a corpo directory')
      dir = parent
    }
  }
}
```

### Generating file IDs

```ts
import { randomUUID } from 'node:crypto'

export const generateId = () => randomUUID().replace(/-/g, '')
```

### Reading and writing config

```ts
export async function readConfig(root: string) { … }
export async function writeConfig(root: string, config: unknown) { … }
```

### Navigation helpers (`src/lib/config.ts`)

Four pure functions for manipulating the navigation tree without reading/writing disk:

```ts
// Remove a file ID from anywhere in the tree
navRemove(nodes: NavNode[], id: string): NavNode[]

// Insert a file ID at a given group path, creating groups as needed
navInsert(nodes: NavNode[], id: string, groupPath: string[]): NavNode[]

// Return true if the ID exists anywhere in the tree
navContains(nodes: NavNode[], id: string): boolean

// Parse "Product/Features/CLI" → ["Product", "Features", "CLI"]
// Empty string → [] (root level)
parseGroupPath(path: string): string[]
```

Used by `corpo new --group` and `corpo mv` to atomically create/move a file
in the navigation tree. `navInsert` recurses into the tree, creating
`{ group, children: [] }` nodes wherever a path segment doesn't yet exist.

## Entry Point

```ts
// src/index.ts
import { Cli } from 'incur'
import { init } from './commands/init.ts'
import { serve } from './commands/serve.ts'
import { newFile } from './commands/new.ts'
import { reply } from './commands/reply.ts'
import { threads } from './commands/threads.ts'
import { lint } from './commands/lint.ts'
import { resolve } from './commands/resolve.ts'
import { mv } from './commands/mv.ts'
import { update } from './commands/update.ts'

Cli.create('corpo', {
  description: 'Git-backed, agent-native file protocol.',
})
  .command('init', init)
  .command('serve', serve)
  .command('new', newFile)
  .command('reply', reply)
  .command('threads', threads)
  .command('lint', lint)
  .command('resolve', resolve)
  .command('mv', mv)
  .command('update', update)
  .serve()
```
