# Overview

`corpo comment <fileId> <line> <body>` inserts a new thread on a corpo file
at a specific line, atomically updating both the body anchor and the frontmatter
thread entry. It is the programmatic equivalent of highlighting a line in the GUI
and typing a comment. The primary consumer is the `review` skill, which currently
has to manually edit frontmatter and body in sync — an error-prone two-step that
`corpo comment` reduces to one command.

## Goals

1. Agents can add threads to a corpo file at a specific line without manually
   editing frontmatter and body
2. The command is atomic — either both the anchor and the frontmatter entry are
   written, or neither is (no lint failures)

## Non-Goals

- Replying to existing threads — use `corpo reply` for that
- Resolving threads — use `corpo resolve`
- GUI-based commenting — this command is CLI/agent-facing only

## Other Requirements

- Generates a new 8-char random hex thread ID
- Inserts `<!-- thread:{id} -->` on a new line immediately after line `<line>`
  in the body
- Adds the thread to frontmatter with `author` set to the invoking identity,
  `timestamp` set to now, and `body` set to the provided comment text
- Runs the equivalent of `corpo lint` internally and errors without writing if
  the result would be invalid
- `<line>` is 1-based; inserting after the last line of the file is valid

# Technical Design

## CLI Interface

```
corpo comment <fileId> <line> "<body>"
```

Arguments:
- `fileId` — 32-char hex corpo file ID
- `line` — 1-based integer line number; anchor is inserted after this line
- `body` — comment text (quoted string)

## Implementation

1. Read `.corpo/files/<fileId>.md`
2. Parse frontmatter and body
3. Generate a new 8-char random hex thread ID (collision-check against existing IDs in the file)
4. Split body on `\n`, insert `<!-- thread:{id} -->` after index `line - 1`
5. Add thread entry to frontmatter `threads` map
6. Re-serialize and write back atomically
7. Print the generated thread ID to stdout so callers can reference it
