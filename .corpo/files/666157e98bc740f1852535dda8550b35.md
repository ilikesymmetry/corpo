Appends a reply to an existing thread. Author is always `claude`. Reads
and writes the file directly on disk — no server required.

## Invocation

```sh
bun src/index.ts reply <fileId> <threadId> "<body>"
```

## Behavior

1. Reads `.corpo/files/{fileId}.md`
2. Parses frontmatter to find `threads.{threadId}`
3. Appends `{ author: 'claude', timestamp: <now>, body }` to `thread.replies`
4. Writes the updated file to disk

Does not create a git commit. The change is a plain file write; the author or
surrounding workflow commits when appropriate.

## Design Intent

The cheapest possible way for an agent to acknowledge or respond to a thread
without opening a server or browser. The `resolve-threads` skill uses this
command after making a content change to confirm the work is done.
