title: "corpo resolve"
description: "Resolve a thread by removing its frontmatter entry and body anchor from a corpo file. CLI variant — plain file write, no auto-commit. For auto-commit behavior, use the Thread Resolution API endpoint."
---

Removes a thread from a corpo file. Two things must be removed together to
maintain anchor integrity: the `threads.{threadId}` entry in frontmatter and
the `<!-- thread:{threadId} -->` anchor in the body. This command removes both
in a single write.

## Invocation

```sh
bun src/index.ts resolve <fileId> <threadId>
```

## Behavior

1. Reads `.corpo/files/{fileId}.md`
2. Deletes `meta.threads[threadId]`
3. Removes `<!-- thread:{threadId} -->` (and trailing newline) from the body
4. If `threads` is now empty, removes the key from frontmatter entirely
5. Writes the updated file to disk

No git commit is created. The CLI `resolve` command is a plain file write —
consistent with `corpo reply`. For auto-commit behavior, use the API endpoint:
`DELETE /api/files/:fileId/threads/:threadId`, which calls `LocalAdapter` and
commits automatically.
