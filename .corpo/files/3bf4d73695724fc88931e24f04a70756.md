The file view is the primary reading surface. It parses raw file content from
the API and renders it in two zones: a metadata header and a markdown body.

## Parsing

The client parses raw `.md` files directly — no server-side processing:

- Splits on `---` frontmatter delimiters
- Parses YAML with **js-yaml** to extract `title`, `description`, `threads`
- Renders the body with **markdown-it** (`typographer: true`, source maps enabled)

`gray-matter` was originally used but is not browser-compatible (depends on
Node's `Buffer`). js-yaml with a custom delimiter split is the correct approach
for browser parsing.

## Header

Displays `title` as an `<h1>` and `description` as a subtitle paragraph.
Raw YAML is never shown to the reader.

## Body

Rendered with markdown-it + `@tailwindcss/typography` prose styles. Thread
anchors (`<!-- thread:{id} -->`) are stripped from the rendered output before
display. Code blocks use highlight.js for syntax highlighting.

### Source Maps

markdown-it's `source_map` core rule stamps every rendered block element with
`data-line` (start line) and `data-line-end` (exclusive end line) attributes.
These are used in two places:

- **Heading-to-thread mapping** — `parseHeadingThreads()` returns a
  `Record<lineNumber, threadId[]>` keyed on source line; `MarkdownRenderer`
  uses `el.dataset.line` to look up which threads a heading carries
- **Anchor placement** — on text selection, the code walks up the DOM from the
  cursor to read `data-line-end`, giving the insertion line for new thread anchors
