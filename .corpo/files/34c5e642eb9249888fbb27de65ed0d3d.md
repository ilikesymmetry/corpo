# Session 5 ‚Äî 2026-03-10

FRD authoring and review loop for Clone/Pull/Push, followed by new CLI
commands, lint rules, GUI improvements, and skills infrastructure.

### Clone/Pull/Push FRD (`4744da1`)

Wrote and iterated a Feature Requirements Doc for the `corpo clone / pull / push`
primitive ‚Äî a git-style snapshot model replacing the `remote_files` live-fetch
pattern. Multiple rounds of thread resolution narrowed scope to two v1 adapters:
`path:` (local files) and `git:` (GitHub via Contents API). Key design decisions
locked in:

- `corpo clone` always creates a new file; errors if the URI is already tracked
- `corpo pull` is the sole re-sync path
- For corpo sources, the full raw source file (frontmatter and all) is used as
  the body ‚Äî no fenced YAML wrapping, pull is identical for all source types
- Push to corpo targets strips local-only fields (`remote`, `syncedAt`,
  `checksum`) before writing back

Removed Google Docs and general web adapters from v1 scope entirely.

### `corpo comment` command (`8b6bf22`)

Added `packages/cli/src/commands/comment.ts` ‚Äî atomically inserts a
`<!-- thread:{id} -->` anchor after a specified 1-based line in the body and
adds the matching frontmatter thread entry. Inlines lint validation before
writing; prints the generated thread ID to stdout. Registered in `index.ts`.

Also created a companion FRD (`7be90c3`) and added it to navigation under
Features/CLI.

### `word-corpus` lint rule (`7597324`)

Added a new rule to `packages/cli/src/commands/lint.ts` that flags any use of
the word "corpus" in corpo file bodies (outside fenced code blocks) and in
frontmatter text fields (title, description, thread bodies, replies). Reports
the field or 1-based body line number. Ran the style skill to clear all
existing violations across 38 files ‚Äî replacements: `corpus file` ‚Üí `corpo
file`, `the corpus` ‚Üí `corpo`.

### Sidebar section navigation (`ecd3fff`)

The navigation sidebar now shows H1‚ÄìH3 headings under the active file when
the file link is clicked while already active. Clicking a heading entry smooth-
scrolls the article to that section via a forwarded ref (`FileViewHandle`) on
`FileView`.

Thread highlight support was also broadened from headings-only to any block
immediately preceding a thread anchor. Renamed `parseHeadingThreads` ‚Üí
`parseBlockThreads` (anchor-line keyed) and updated CSS classes from
`threaded-heading` to `threaded-block`.

### Skills infrastructure (`58a7616`, `e8b7555`)

Added three local skills under `skills/`:

- **`frd/`** ‚Äî write or update a Feature Requirements Doc
- **`review/`** ‚Äî design pass that reads a corpo file end-to-end, identifies
  3‚Äì6 substantive findings, and adds each as a thread via `corpo comment`
- **`style/`** ‚Äî iteratively runs `corpo lint`, fixes all flagged issues, and
  relints until clean
- **`journal/`** (renamed from `doc-sync/`) ‚Äî updated to commit outstanding
  changes with user consent, store HEAD SHA as `commit` in frontmatter, and
  scope `git log` to the range between the previous journal's `commit` and
  current HEAD
