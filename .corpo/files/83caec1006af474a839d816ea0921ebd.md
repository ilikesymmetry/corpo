# Session 3 — 2026-03-09

Fourth block. Focused on distribution, DX polish, and agent-friendly CLI primitives.

### Published to npm as @ilikesymmetry/corpo 0.1.0

Attempted to publish as `corpo` — blocked by npm similarity policy (too close
to `cors` / `copy`). Switched to scoped package `@ilikesymmetry/corpo` with
`--access=public`. `prepublishOnly` runs `build:gui` automatically so the GUI
dist is always bundled. Updated README install command accordingly.

### viewer → gui rename

Renamed `packages/viewer/` to `packages/gui/` across the entire codebase:
directory, package name, build scripts, `.gitignore`, `.npmignore`, path refs
in the server, and all ~18 corpo feature files. "Viewer" was generic;
"GUI" maps to the 3-interface framing (CLI / API / GUI).

### Bun install onboarding

Two touchpoints improved:
- **README**: Bun install is now step 1 with a runnable `curl` command before
  the npm install step.
- **`bin/corpo.js` error**: When bun is not found, the shim now prints both
  `curl` and `npm install -g bun` as runnable alternatives (npm is always
  available since Node is guaranteed present in an npm install context).

### Corpo skill self-documentation

Updated `skills/corpo/SKILL.md` to be more self-contained for agents:
added package identity (`@ilikesymmetry/corpo`, Bun ≥ 1.0), Step 1 CLI
availability check with full install instructions, source-repo invocation path,
`--help` progressive disclosure note, all 7 commands in the table, and a
lint-after-edit reminder.

### Navigation placement: corpo new --group and corpo mv

`corpo new` now accepts an optional `--group <path>` flag to atomically create
a file and insert it into the navigation tree in one command. `corpo mv <id>
<path>` is a new command to move an existing file to a different navigation
group. Both use `/`-delimited group labels. Missing groups are created
automatically. Moving to root uses an empty string.

Four pure nav helpers added to `src/lib/config.ts`:
`navInsert`, `navRemove`, `navContains`, `parseGroupPath`.

### corpo update

New command. Fetches `registry.npmjs.org` for the latest version, compares
with the version embedded in `package.json`, and runs `npm install -g
@ilikesymmetry/corpo@latest` if outdated. No-ops with a message when current.

### corpo serve --detach

Added `--detach` flag to `corpo serve` for non-blocking agent use. When set,
spawns a background server process via `Bun.spawn` with `detached: true` using
`process.execPath` + `process.argv[1]` (works in both source and installed
contexts), unrefs the child, and returns `{ url, pid, detached: true }`
immediately. Default blocking behavior unchanged for human terminal use.

### Dev scripts: bun run dev and bun run dev:bg

Added two convenience scripts to the root `package.json`:
- `dev`: runs `dev:cli & dev:gui & wait` — both servers in one terminal,
  Ctrl+C kills both. Human use.
- `dev:bg`: uses `corpo serve --detach` for the API + `nohup ... &` for Vite,
  returns immediately with PIDs. Agent use (avoids blocking the Bash tool).
  Vite logs go to `/tmp/corpo-gui.log`.
