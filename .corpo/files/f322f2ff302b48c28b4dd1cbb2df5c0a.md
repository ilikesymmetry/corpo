---
title: "corpo lint"
description: "Check thread anchor integrity across one or all corpus files. Validates 1:1 correspondence between frontmatter threads and body anchors. Also runs automatically as a Claude Code PostToolUse hook on every corpus file edit."
---

Validates that every thread in frontmatter has a corresponding anchor in the
body, and vice versa. Two violation types:

- **orphaned-thread** — `threads.{id}` in frontmatter with no matching
  `<!-- thread:{id} -->` in the body
- **dangling-anchor** — `<!-- thread:{id} -->` in the body with no matching
  `threads.{id}` in frontmatter

## Invocation

```sh
bun src/index.ts lint [fileId]   # one file, or all corpus files if omitted
```

Exits with code 1 and prints violations to stderr if any are found.

## False Positive Prevention

Code fences are stripped before scanning for anchors. Thread IDs appearing
inside backtick code blocks (e.g., documentation examples) are not counted as
anchors.

## PostToolUse Hook

Registered in `.claude/settings.json` to run automatically after every Edit
or Write tool call on a `.corpo/files/*.md` file. The hook (`scripts/lint-hook.py`)
reads the modified file path from the tool input JSON on stdin, extracts the
file ID, and runs `corpo lint <fileId>`. If violations are found, the hook exits
1 and the error surfaces in the Claude Code conversation immediately — before
any git commit.

This makes anchor integrity failures impossible to miss.
