Any text selected in the file body triggers a floating comment input. On
submit, a new thread is anchored to the selected position and saved.

## Interaction

1. User selects text in the article
2. A floating `CommentInput` appears just below the selection bounding rect
3. User types a comment and presses Enter (Shift+Enter for newlines)
4. Input dismisses; the thread appears in the panel and the heading gains an
   amber highlight

## Thread Creation

On submit:

1. An 8-char hex thread ID is generated (`generateThreadId()`)
2. A `Thread` object is created: `{ author: 'me', timestamp: now, body }`
3. The thread is added to `localMeta.threads` in local state
4. `<!-- thread:{id} -->` is inserted into the body at the correct source line
5. The file is committed via `PUT /api/files/:id/commit`

## Anchor Placement

The anchor must be inserted by source line number, not text content. Text
matching fails because rendered DOM text doesn't match raw markdown (tables,
blockquotes, typographer quote conversions).

**Solution:** markdown-it stamps `data-line-end` on every rendered block element.
On `mouseup`, the code walks up the DOM from `range.endContainer` to find the
nearest `data-line-end` attribute. That line number is where the anchor is
inserted in the raw source.

This logic is unit tested in `gui/src/lib/serialize.test.ts`.

## Dismissal

A document-level `mousedown` listener clears the selection and hides the input
whenever the user clicks outside the `CommentInput` element (detected via
`data-comment-input` attribute).
