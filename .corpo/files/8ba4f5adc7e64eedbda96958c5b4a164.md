---
title: "Navigation Sidebar"
description: "Recursive sidebar rendered from the navigation tree returned by GET /api/navigation. Supports nested file/group trees, collapsible groups, lazy file title resolution, and a hover-reveal collapse button."
---

The sidebar renders the full corpus navigation tree on mount and stays static
while the user navigates between files.

## Data Flow

1. On mount, fetches `GET /api/navigation`
2. Traverses the recursive `Node[]` tree to render groups and file links
3. File titles resolve lazily: each file ID triggers a `GET /api/files/:id`
   to extract `sidebarTitle` or `title` from frontmatter. The raw ID is shown
   as a fallback while the title loads.

## Group Behavior

Groups are collapsible. Each group renders with a chevron (▾ open, ▸ closed)
and a label. Clicking toggles all children. Groups can be freely nested.
Chevrons are wrapped in `text-base` so they render larger than the surrounding
`text-xs` label text.

## Collapse Control

A collapse/expand button appears on the right edge of the sidebar when the user
hovers over it. Clicking collapses the sidebar to zero width via smooth CSS
transition. The button arrow flips to signal the expand direction.

State lives in `App.tsx` and is passed down as `collapsed` and `onToggle` props.
The button is absolutely positioned and never causes layout shift.
