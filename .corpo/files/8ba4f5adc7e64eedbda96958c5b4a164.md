The sidebar renders the full navigation tree on mount and stays static
while the user navigates between files.

## Data Flow

1. On mount, fetches `GET /api/navigation`
2. Traverses the recursive `Node[]` tree to render groups and file links
3. File titles resolve lazily: each file ID triggers a `GET /api/files/:id`
   to extract `sidebarTitle` or `title` from frontmatter. The raw ID is shown
   as a fallback while the title loads.

## Group Behavior

Groups are collapsible. Each group renders with a chevron (▾ open, ▸ closed)
and a label. Clicking toggles all children. Groups can be freely nested.
Chevrons are wrapped in `text-base` so they render larger than the surrounding
`text-xs` label text.

## Collapse Control

Collapse is handled by the base `Sidebar` component — a full-height handle
strip on the right edge of the sidebar. See Layout Controls for the handle
strip mechanics.

State lives in `App.tsx` as `sidebarCollapsed`, passed down as `collapsed`
and `onCollapsedChange` props to `NavigationSidebar`.
