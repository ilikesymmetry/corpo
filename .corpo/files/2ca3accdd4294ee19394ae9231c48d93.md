---
title: "corpo mv"
description: "Move a corpo file to a different navigation group. Takes a file ID and a slash-delimited group path (e.g. 'Product/Features/CLI'). Creates groups that don't exist. An empty path moves the file to the root navigation level. No-ops with a message if the file is already in the target group."
---

Moves a file's position in the navigation tree without touching the file on disk.
Only `.corpo/config.json` is modified.

## Invocation

```bash
# Move to an existing group
bun src/index.ts mv <id> "Product/Features/CLI"

# Move to a new nested group (created automatically)
bun src/index.ts mv <id> "Research/Scratch"

# Move to the root navigation level
bun src/index.ts mv <id> ""
```

**Output (moved):**
```json
{ "id": "a1b2...", "group": "Product/Features/CLI", "moved": true }
```

**Output (no-op):**
```json
{ "id": "a1b2...", "group": "Product/Features/CLI", "moved": false, "message": "Already in that group." }
```

## Behavior

- **File not found:** throws an error before touching config.
- **Group doesn't exist:** creates all missing groups along the path.
- **Empty group path:** moves to root (appended to the top-level navigation array).
- **Already in target:** no-op with `moved: false` and a message.
- **File not yet in navigation:** inserts at target without attempting removal.

## Group Path Syntax

`/`-delimited group labels matching the `group` keys in `.corpo/config.json`.
Labels are matched by string equality. Leading/trailing whitespace is trimmed.

```
"Product"                  → top-level Product group
"Product/Features/CLI"     → CLI subgroup inside Features inside Product
""  or  "/"               → root navigation level
```

## Pair with `corpo new`

```bash
# Create and place atomically:
corpo new "Title" "Desc" --group "Product/Features/CLI"

# Or create first, place later:
ID=$(corpo new "Title" "Desc" --filter-output id)
corpo mv $ID "Product/Features/CLI"
```
