# API Server Implementation

The corpo server is a Hono app running on Bun. It is intentionally thin: each
route delegates to a `LocalAdapter` that reads and writes the local filesystem.
No business logic lives in the routes themselves.

See [Server API](d8b4f3e2a1c54f9bb2e8c1d6f3e5b7a2) for the endpoint contract
and [CLI Stack](f1a2b3c4d5e64f7a8b9c0d1e2f3a4b5c) for how the server is
started from `corpo serve`.

---

## Dev Setup

Prerequisites: [Bun](https://bun.sh) installed.

```sh
# Install dependencies
bun install

# Run the server directly against a local corpus (no compile step)
bun run src/index.ts serve --port 3000
```

The server reads `.corpo/config.json` by walking up from the current directory.
Run from inside any corpus (a directory containing `.corpo/`) or pass the
corpus root explicitly if you extend the command to accept one.

To iterate on the server without rebuilding the binary, `bun run` is sufficient.
The binary compile step (`bun build --compile`) is only needed for distribution.

## LocalAdapter

The adapter implements the `StorageAdapter` interface and is the only place
that touches the filesystem. The Hono routes call adapter methods; they do
not call `fs` directly.

```ts
import { readFile, writeFile, unlink, access } from 'node:fs/promises'
import { join } from 'node:path'
import type { Navigation, Node } from './types.ts'

export class LocalAdapter {
  constructor(private root: string) {}

  private get configPath() {
    return join(this.root, '.corpo', 'config.json')
  }

  private filePath(id: string) {
    return join(this.root, '.corpo', 'files', `${id}.md`)
  }

  // GET /api/navigation
  async getNavigation(): Promise<Navigation> {
    const raw = await readFile(this.configPath, 'utf8')
    return JSON.parse(raw).navigation
  }

  // PUT /api/navigation
  async putNavigation(navigation: Navigation): Promise<void> {
    const raw = await readFile(this.configPath, 'utf8')
    const config = JSON.parse(raw)
    await writeFile(this.configPath, JSON.stringify({ ...config, navigation }, null, 2))
  }

  // GET /api/files/:id
  async getFile(id: string): Promise<string | null> {
    try {
      return await readFile(this.filePath(id), 'utf8')
    } catch (e: any) {
      if (e.code === 'ENOENT') return null
      throw e
    }
  }

  // POST /api/files
  async createFile(raw: string): Promise<string> {
    const id = randomUUID().replace(/-/g, '')
    await writeFile(this.filePath(id), raw)
    return id
  }

  // PUT /api/files/:id/commit
  async commitFile(id: string, raw: string): Promise<void> {
    await access(this.filePath(id)) // throws ENOENT if missing → caught by route
    await writeFile(this.filePath(id), raw)
  }

  // DELETE /api/files/:id
  // Removes local file or remote_files entry, then strips ID from navigation.
  async deleteFile(id: string): Promise<void> {
    const raw = await readFile(this.configPath, 'utf8')
    const config = JSON.parse(raw)

    let found = false

    // Local file
    try {
      await unlink(this.filePath(id))
      found = true
    } catch (e: any) {
      if (e.code !== 'ENOENT') throw e
    }

    // Remote file entry
    if (config.remote_files?.[id]) {
      delete config.remote_files[id]
      found = true
    }

    if (!found) throw Object.assign(new Error('File not found'), { code: 'NOT_FOUND' })

    // Strip ID from navigation tree
    config.navigation = removeFromNavigation(config.navigation, id)
    await writeFile(this.configPath, JSON.stringify(config, null, 2))
  }
}
```

## Navigation Cleanup

`removeFromNavigation` is a recursive traversal that filters out a file ID
wherever it appears in the `Node[]` tree. Called by `deleteFile` so the
caller never has to do it manually.

```ts
type Node = string | { group: string; children: Node[] }

function removeFromNavigation(nodes: Node[], id: string): Node[] {
  return nodes
    .filter(node => node !== id)
    .map(node => {
      if (typeof node === 'string') return node
      return { ...node, children: removeFromNavigation(node.children, id) }
    })
}
```

## Hono Routes

Routes are thin wrappers over the adapter. Error handling is centralized
via Hono's `onError` hook so individual routes stay clean.

```ts
import { Hono } from 'hono'
import { LocalAdapter } from './adapter.ts'

export function createServer(root: string) {
  const app = new Hono()
  const adapter = new LocalAdapter(root)

  app.onError((err, c) => {
    const status = err.code === 'NOT_FOUND' ? 404 : 500
    return c.json({ error: err.message }, status)
  })

  app.get('/api/navigation', async (c) => {
    return c.json({ navigation: await adapter.getNavigation() })
  })

  app.put('/api/navigation', async (c) => {
    const { navigation } = await c.req.json()
    if (!navigation) return c.json({ error: 'missing navigation' }, 400)
    await adapter.putNavigation(navigation)
    return c.body(null, 204)
  })

  app.get('/api/files/:id', async (c) => {
    const file = await adapter.getFile(c.req.param('id'))
    if (file === null) return c.json({ error: 'file not found' }, 404)
    return c.text(file)
  })

  app.post('/api/files', async (c) => {
    const { raw } = await c.req.json()
    if (!raw) return c.json({ error: 'missing raw' }, 400)
    const id = await adapter.createFile(raw)
    return c.json({ id }, 201)
  })

  app.put('/api/files/:id/commit', async (c) => {
    const { raw } = await c.req.json()
    if (!raw) return c.json({ error: 'missing raw' }, 400)
    await adapter.commitFile(c.req.param('id'), raw)
    return c.body(null, 204)
  })

  app.delete('/api/files/:id', async (c) => {
    await adapter.deleteFile(c.req.param('id'))
    return c.body(null, 204)
  })

  // Viewer SPA — must be last
  app.get('*', serveViewer())

  return app
}
```

## Serving the Viewer

The viewer `dist/` is embedded in the CLI binary at compile time. At runtime,
`serveViewer()` returns it from memory — no disk read.

```ts
import index from '../../viewer/dist/index.html' with { type: 'text' }
import { serveStatic } from 'hono/bun'

function serveViewer() {
  return async (c: Context) => {
    // All routes return the SPA shell; client-side routing handles the rest
    return c.html(index)
  }
}
```

Hashed static assets (JS, CSS) are handled by Bun's asset embedding — Bun
resolves `import` statements for binary assets at compile time and maps them
to content-addressed paths automatically.
