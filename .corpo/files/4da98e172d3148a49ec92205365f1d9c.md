Creates a new corpus file with the minimum required frontmatter. Outputs the
generated file ID and its path so the author knows where to write the body.

## Invocation

```sh
bun src/index.ts new "<title>" "<description>"
```

**Output:**
```json
{ "id": "a1b2c3d4...", "path": ".corpo/files/a1b2c3d4....md" }
```

## What It Creates

```md
---
title: "My New File"
description: "What this file is about."
---
```

The body is empty. The author fills it in after creation, either directly
or by opening the file in `corpo serve`.

## Shared Internals

Both `corpo new` and `POST /api/files` call the same `createCorpoFile(root, raw)`
function from `src/lib/files.ts`. That function generates a UUID v4 (hyphens
stripped) via `generateId()` and writes the raw content to
`.corpo/files/{id}.md`. Neither the CLI nor the API duplicates this logic.

```
corpo new         → createCorpoFile()  ← POST /api/files (LocalAdapter)
```

## Navigation

The new file is **not** automatically added to the navigation tree. Add it
manually to `.corpo/config.json` under the appropriate group, or use
`PUT /api/navigation` from a client.

## Implementation

`src/commands/new.ts`
`src/lib/files.ts` — `createCorpoFile()`
`src/lib/id.ts` — `generateId()`
