Creates a new corpo file with the minimum required frontmatter. Outputs the
generated file ID and its path so the author knows where to write the body.

## Invocation

```bash
# Create only
bun src/index.ts new "<title>" "<description>"

# Create and place in navigation
bun src/index.ts new "<title>" "<description>" --group "Product/Features/CLI"

# Place at root navigation level
bun src/index.ts new "<title>" "<description>" --group ""
```

**Output:**
```json
{ "id": "a1b2c3d4...", "path": ".corpo/files/a1b2c3d4....md", "group": "Product/Features/CLI" }
```

`group` is `undefined` in the output when `--group` is omitted.

## What It Creates

```md
---
title: "My New File"
description: "What this file is about."
---
```

The body is empty. The author fills it in after creation, either directly
or by opening the file in `corpo serve`.

## Shared Internals

Both `corpo new` and `POST /api/files` call the same `createCorpoFile(root, raw)`
function from `src/lib/files.ts`. That function generates a UUID v4 (hyphens
stripped) via `generateId()` and writes the raw content to
`.corpo/files/{id}.md`. Neither the CLI nor the API duplicates this logic.

```
corpo new         → createCorpoFile()  ← POST /api/files (LocalAdapter)
```

## Navigation Placement

Pass `--group "<path>"` to atomically create the file and insert it into the
navigation tree. The path is `/`-delimited group labels (e.g. `"Product/Features/CLI"`).
Groups that don't exist are created automatically. An empty string places the
file at the root navigation level.

Without `--group`, the file is created but not added to navigation. Use
`corpo mv <id> <group>` to place it later.
