# Finalize README and docs for external audience

Polish all public-facing documentation for the launch. The repo is the landing
page — it needs to be compelling and complete for a developer seeing Corpo for
the first time.

## Scope

- **README**: update to reflect the shipped CLI with real install instructions,
  a quickstart (3 commands to get to a published link), and a clear value prop
- **docs/protocol.md**: ensure the spec reflects all Phase 1 decisions exactly
- **docs/user-flows.md**: update to match the actual shipped CLI behavior
- **docs/migration.md**: add real adapter documentation for Google Docs
- Remove or archive discovery-phase files that shouldn't be public-facing
  (or keep them — they tell a good product story)

## Done when

- [ ] README has working install instructions and a quickstart that actually
      runs end-to-end
- [ ] Protocol spec is accurate and complete as of launch
- [ ] No placeholder or TBD content in any public-facing doc
- [ ] Docs read cleanly to an external developer with no prior context
